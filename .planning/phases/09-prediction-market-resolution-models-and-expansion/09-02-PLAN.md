  <behavior>
    CoinGeckoClient:
    - get_price("bitcoin") returns float > 0 (live) or 0.0 when env COINGECKO_OFFLINE=true
    - get_price_change_7d("bitcoin") returns float (positive or negative) or 0.0 offline
    - Uses httpx.get() (synchronous — not async; feature fetch at training time is synchronous)
    - Base URL: https://api.coingecko.com/api/v3/simple/price?ids={coin_id}&vs_currencies=usd&price_change_percentage=7d
    - On any httpx exception: returns (0.0, 0.0) — never raises

    FECClient:
    - get_polling_average(race_id: str) -> float: calls https://api.open.fec.gov/v1/presidential/coverage_end_date/ (public, no key required for basic data); returns average of latest polling margins normalized to [0, 1]; offline (FEC_OFFLINE=true) returns 0.0
    - get_election_proximity_days(election_date_str: str) -> int: parses date string "%Y-%m-%d", returns max(0, (election_date - today).days); no network call needed
    - On any exception: returns safe defaults (0.0, 365) — never raises

    BLSClient:
    - Hardcoded release calendar for CPI, PPI, NFP, GDP (monthly cadence): use a static dict `RELEASE_CADENCE_DAYS = {"CPI": 30, "PPI": 30, "NFP": 30, "GDP": 90}` (BLS calendar is complex to parse; static cadence is sufficient for feature engineering)
    - get_days_since_last_release(series: str) -> int: today.day / RELEASE_CADENCE_DAYS[series] * cadence (approx); offline (BLS_OFFLINE=true) returns 30
    - get_is_release_imminent(series: str, threshold_days: int = 3) -> bool: returns get_days_since_last_release(series) >= (cadence - threshold_days)
    - On unknown series: returns (30, False)

    All 3 clients: keep each file under 120 lines. No new dependencies — use httpx (already in workspace) and stdlib datetime only.
